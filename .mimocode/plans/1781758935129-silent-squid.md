# Implementation Plan: UI Optimization with Miuix Components

## Goal
Replace all custom/hand-rolled UI elements in `App.kt` with Miuix library components for a unified HyperOS design style.

## Current State
The app currently uses only 3 Miuix components: `Card`, `Scaffold`, `Text`. Everything else is hand-rolled with raw `Modifier.background()`, `BasicTextField`, custom progress bars, etc.

## Miuix Components to Use

| Current UI | Miuix Replacement | Package |
|---|---|---|
| Custom `TopBar` with `Row` | `TopAppBar` | `top.yukonga.miuix.kmp.basic.TopAppBar` |
| Custom `TabBar` with `Row` + `Text` | `TabRow` | `top.yukonga.miuix.kmp.basic.TabRow` |
| `BasicTextField` in `EditableField` | `TextField` | `top.yukonga.miuix.kmp.basic.TextField` |
| `Text(...).clickable` as buttons | `Button` | `top.yukonga.miuix.kmp.basic.Button` |
| Custom `Box` progress bar | `LinearProgressIndicator` | `top.yukonga.miuix.kmp.basic.LinearProgressIndicator` |
| Custom `AddDeviceDialog` bottom sheet | `OverlayBottomSheet` | `top.yukonga.miuix.kmp.basic.OverlayBottomSheet` |
| Custom sidebar overlay | `OverlayDialog` or keep as-is (sidebar is non-standard) | |
| `Text("×")` close button | `IconButton` | `top.yukonga.miuix.kmp.basic.IconButton` |
| Custom SAR/version toggles | `Switch` or `Checkbox` | `top.yukonga.miuix.kmp.basic.Switch` |
| Log entries with `Text` | Keep as-is (custom log format) | |
| `Spacer` dividers | `Divider` | `top.yukonga.miuix.kmp.basic.Divider` |
| Section headers | `SmallTitle` | `top.yukonga.miuix.kmp.basic.SmallTitle` |

## Files to Modify

**Single file: `shared/src/commonMain/kotlin/com/miband/app/App.kt`** (~807 lines)

## Implementation Steps

### Step 1: TopBar → TopAppBar
Replace the custom `TopBar` composable with Miuix `TopAppBar`:
```kotlin
TopAppBar(title = { Text("BANDBURG") }, actions = {
    IconButton(onClick = onMenuClick) { Icon(...) }
})
```

### Step 2: TabBar → TabRow
Replace custom `TabBar` with Miuix `TabRow`:
```kotlin
TabRow(selectedTabIndex = activeTab) {
    Tab(selected = activeTab == 0, onClick = { activeTab = 0 }, text = { Text("表盘") })
    Tab(selected = activeTab == 1, onClick = { activeTab = 1 }, text = { Text("应用") })
    Tab(selected = activeTab == 2, onClick = { activeTab = 2 }, text = { Text("安装") })
}
```

### Step 3: EditableField → TextField
Replace `BasicTextField`-based `EditableField` with Miuix `TextField`:
```kotlin
TextField(value = value, onValueChange = onValueChange, label = { Text(label) }, placeholder = { Text(placeholder) })
```

### Step 4: Action Buttons → Button
Replace `Text(...).clickable` patterns with Miuix `Button`:
- "保存设备" / "取消" → `Button` / `SecondaryButton` (or `Button` with `outlined` style)
- "开始安装" → `Button`
- "选择文件" → `Button`
- "断开连接" / "连接设备" → `Button`
- "设为当前" / "卸载" / "启动" / "删除" → smaller `Button` or `TextButton`

### Step 5: Progress Bar → LinearProgressIndicator
Replace custom `Box` progress bar:
```kotlin
LinearProgressIndicator(progress = { installProgress }, modifier = Modifier.fillMaxWidth())
```

### Step 6: AddDeviceDialog → OverlayBottomSheet
Replace custom bottom sheet dialog with Miuix `OverlayBottomSheet`:
```kotlin
OverlayBottomSheet(visible = showAddDeviceDialog, onDismiss = { showAddDeviceDialog = false }) {
    // form content
}
```

### Step 7: Version/Connect Type Toggles → Switch or Dropdown
Replace the click-to-toggle text with proper `Switch` or dropdown selectors.

### Step 8: Add Dividers and SmallTitles
Add `Divider` between sections and `SmallTitle` for section headers for better visual hierarchy.

### Step 9: Clean Up
- Remove unused imports (`SolidColor`, `TextStyle`, `BasicTextField`)
- Remove hardcoded colors (`Color(0xFF1A73E8)`, `Color(0xFFE0E0E0)`, etc.) — use `MiuixTheme.colorScheme` instead
- Ensure all spacing/padding follows Miuix conventions
