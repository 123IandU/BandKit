## Fix 1: Auth Flow - Call nativeHandleAuthStep2 (CRITICAL)

**File**: `shared/src/androidMain/.../BandBurgManager.kt`

**Problem**: `handleReceivedPackets` detects DeviceVerify (type=1, id=4) and completes `authDeferred`, but NEVER calls `NativeLib.nativeHandleAuthStep2()` to send AppConfirm back to the device. The device never receives the confirmation, so auth never completes and L2 encryption never activates.

**Reference**: AstroBox-NG `install.rs` shows auth is a prerequisite for all operations.

**Fix** in `handleReceivedPackets()`, when type==1 && id==4:
1. Extract the full packet JSON
2. Find the `authDeviceVerify` or `device_verify` payload object
3. Serialize it back to JSON string
4. Call `NativeLib.nativeHandleAuthStep2(handle, deviceVerifyJson, true)` 
5. Send the returned bytes via outputStream
6. Then complete authDeferred

## Fix 4: File Installation - Correct MASS Protocol

**Files**: `BandBurgManager.kt`, `ProtobufBuilder.kt`

**Current problem**: The `installFile` method sends a generic `buildPrepareInstall(type=19, id=1)` which doesn't match the actual protocol. AstroBox-NG uses type-specific prepare requests.

**Reference** (AstroBox-NG `install.rs:452-537`):

### Watchface Install:
- Prepare request: `WearPacket { type: WatchFace(4), id: PrepareInstallWatchFace, payload: WatchFace(PrepareInfo{id, size, version_code:65536}) }`
- Response: `WatchFace(PrepareStatus{status: Ready})`
- After MASS transfer: `WatchFace(InstallResult{code})` 

### ThirdPartyApp Install:
- Prepare request: `WearPacket { type: ThirdpartyApp(20), id: PrepareInstallApp, payload: ThirdpartyApp(InstallRequest{package_name, version_code:114514, package_size}) }`
- Response: `ThirdpartyApp(InstallResponse{prepare_status: Ready})`
- After MASS transfer: `ThirdpartyApp(InstallResult{code})`

### Firmware Install:
- Prepare request: `WearPacket { type: System(2), id: PrepareOta, payload: System(PrepareOta.Request{force:true, type:All, firmware_version:"99.99.99", file_md5, change_log:"AstroBox Firmware Update"}) }`
- Response: `System(PrepareOtaResponse{prepare_status})`

**Fix**:
1. Add `buildWatchfaceInstallRequest(id, size)`, `buildThirdpartyAppInstallRequest(packageName, size)`, `buildFirmwareInstallRequest(md5, size)` to ProtobufBuilder
2. Update `installFile()` to use type-specific prepare requests
3. Wait for prepare response before sending MASS data
4. Add MD5 calculation (use existing `tools::calc_md5` pattern or Kotlin equivalent)
5. Properly encode MASS data: `comp_data(0x00) | data_type(1B) | md5(16B) | length(4B LE) | data | crc32(4B LE)`
6. Wait for install result response
